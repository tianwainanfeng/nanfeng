Linux: Command Line

Main Shell

Help commands:

  • man
    • manual pages for help
    • example: man man
  • whereis
  • whatis
  • locate

List commands:

  • ls
    • ls -m : list files as a single line, with comma separations
    • ls -x : list files and sort horizontally
    • ls -F : list files and show directories with "/" character appended to filename
    • ls -R : list all files and recursively descends directories
    • ls -dR /*/*/* : show a certain number of directory level
    • li -l : list in a long format
  • dir
    • list directories
  • vdir
    • same as ls -l

Read commands:

  • cat
    • cat -n : output with line number
  • more
    • type "h" for help
    • type "q" for quit
  • less
    • refer more
  • head
    • head -n : show n lines
  • tail
    • refer head

Create commands:

  • touch
    • touch newfile.txt : create a new file; if the file exists, it will update the file's date or time.
    • touch -t 1000110022 myfile.txt : set the time for the file
  • mkdir
    • mkdir -p temp/parent/child : build a heirarchy of directories
  • ln
    • ln : create hard link
    • ln -s : create symbolic link; a symbolic link is merely a pathname or alias to the original file
    • ln file1 file2 : file2 will be the same as file1. Removing file1, file2 wil remain. Changing file1, the changes will apprear in file2
  • mc
    • mc : Midnight Commander, a graphical interface for handing files, a visual shell

Remove commands:

  • rm
    • rm -rf : force to remove recursively
    • rm -i : use the interactive option to check if you want to delete the file
  • rmdir
    • rmdir folder : remove a folder but the folder should be empty first

Moving command

  • mv
    • mv a.txt b.txt : rename the filename
    • mv -b file1 file2 : if file2 exists, create a backup of file2 with a default extention of (~)
    • mv -i : interactive mode
  • cp
    • cp i : interactive mode
    • cp -r : copy recrusively
    • cp -P : copy with the directory structure. One may use tree to check the structure

Searching commands:

  • Regular expressions
    • * : matches any character
    • ? or . : matches a single character
    • [xxx] or [x-x] : matches a character in a range of characters
    • \x : matches a character such as ? or \
    • ^pattern : matches pattern to the beginning of a line
    • $pattern : matches pattern to the end of a line
  • grep
    • grep ^[0-9] : find all lines that begin with a number
    • grep "(b)" : search for pattern of the letter b in parentheses
    • grep -n : display the line number
    • grep -nf myword.txt otherfile.txt: myword.txt contains all the words that we want to search, which are saved in a file
    • grep -F : same as fgrep
    • grep -E : same as egrep
  • egrep
    • egrep ^[0-9] : same as grep ^[0-9]
    • egrep "\([b]\)" : search for pattern of the letter b in parentheses, same as grep "(b)"
    • egrep -n : show line number
  • fgrep
    • fgrep : find fixed string

Compressing and uncompressing commands:

  • tar
    • tar : tape archive program
    • tar -c mydir > mydir.tar : create a tar archive; mydir will be still there
    • tar -cv mydir > mydir.tar : create a tar archive; mydir will be removed; also, what directories and files are being added will be shown.
    • tar -cv mydir > mydir.tar : have tar ask if you want to add each file
    • tar tf mydir.tar : -t to list an archive's contents; -f> to specify a tar archive to use
    • tar -xvf mydir.tar : to extract everything
    • tar -xvwf mydir.tar : to extract selected files
    • tar -xf mydir.tar mydir/test.txt : to extract a single file; tar will overide files with the same name
  • cpio
    • cpio : copy files in and out of tar or cpio archives
  • gzip
    • gzip : compress files; save disk space; combine with tar to deal with .\tgz or .tar.gz format
    • gzip mydir.tar : compress a file or an archive; append a .gz extension; delete the original file
    • gzip -d or gunzip mydir.tar.gz : uncompress file; file should have a .gz (.Z, -gz, .z, -z, or _z) extension
    • gzip -S : specify your own extension
  • compress
    • compress file : compress a file; add a .Z extension.
    • uncompress file.Z: inverse of compress file

System administration commands:

  • su
    • su : superuser command, run a command as any user on the system
    • whoami : check user
  • df
    • : free disk space command, gather and summarize the statistics about all current mounted filesystems
  • du
    • du : disk usage command, summarize how disk is being used
  • free
    • free : show breakdowns fo the amounts and totals of free and used memory, including the swapfile usage
  • vmstat
    • vmstat : a general=purpose monitoring program, offer real-time display of memoery usage, virtual memory statistics, disk activity, system usage, and CPU activity
  • top
    • top : a system monitor that displays statistical information about how Linux is currently handling the memory, swap file and processes
  • uptime
    • uptime : show how long Linux has been running, how many users are on, and three system load average
  • w
    • w : show uptime information and that for each user
  • adduser
    • adduser www: add a user www
  • passwd
    • passwd www : set password for user www
  • chfn
    • chfn www : change finger information of user www; use finger to check the finger information

Ownership and Permission commands:

  • file types
    • b : block device
    • c : character device
    • d : directory
    • b : symbolic link
  • permission flags
    • r : read
    • w : write
    • r : execute (search for a directory)
  • groups
    • owner: first three characters
    • group: the next three characters after owner (use groups to list groups)
    • others: the last three characters after group
  • chmod
    • chmod : change file or directory permissions; it uses octal, or base eight, notation in modifying file or directory permissions. The 4-2-1 sequence corresponds to the three rwx sequences in the permissions flags. chmod also has a command-line form: ugoa +-= rwxXstugo
    • chmod 600 file : only owner can read(4) and write(2). Here 4+2=6
    • chmod 664 file : only owner and group can read(4) and write(2). Here 4+2=6. Others can read(4) only.
    • chmod 700 mydir : only owner can read(4) and write(2) and access(1) to mydir. Here 4+2+1=7
    • chmod go-rwx file : remove the read-, write-, and execute- permissions of group and others for file
    • chmod o+x mydir : give others permission to access mydir
  • chown
    • chown : change ownership command
    • chown user:group file : give user in group the access to file; if user is omitted, all users in group will have access
  • chgrp
    • chgrp : change group command; change group ownerships using group name or group id (GID)
  • newgrp
    • newgrp : shift the current group membership
