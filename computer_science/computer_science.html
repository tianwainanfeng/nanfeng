Data storage

  • bits: short for binary digits, 0s and 1s, that are for encoded information.
  • flip-flop: a collection of circuits built by AND, OR, XOR and NOT gates.
  • main memory: a large collection of circuits that store the bits.
  • cell: a manageable unit that a computer's main memory is organized, e.g, a cell size of 8 bits. A string of 8 bits is called a byte.
  • address: a unique "name" that can be used to identify individual cells in a computer's main memory.
  • Random Access Memory (RAM): main memory, since it is organized as individual, addressable cells that can be accessed independently.
  • dynamic memory: dynamic RAM that uses modern techniques to decrease the response time.
  • mass storage: secondary storage system, additional memory devices to main memory.
  • ASCII: American Standard Code for Information Interchange, use bit patterns of length seven to represent the letter, punctuation symbols, digit number and certain control information; it is extended to an 8 bits to fit conveniently into a typical byte-size memory cell and provide 128 additional bit patterns.
  • Unicode: extention of ASCII by International Organization for Standardization (ISO) of 16 bits to allow text written in other language, such as Chinese and Japanese to be represented.

Data manipulation

  • Central Processing Unit (CPU): processor, circuitry in a computer that controls the manipulation of data. A CPU consistis of three parts: arithmetic/logic unit (perform operations), control unit (coordinates the machine's activities), and registers (temporaray storage).
  • pipelining: technique of allowoing teh steps in the machine cycle to overlap, it can increase the execution speed. While one instruction is being fexecuted, the net instruction can be fetched. More than one instruction are in "the pipe" at one time.
  • Multi-Core CPU: more than one CPUs residing on the same chip along with shared cache memory.
  • parallel processing: perform several activities at the same time; pipelining can be viewed as a first step.
  • batch processing: execution of jobs by collecting them in a single batch, then executing them withouth further interaction with the user. Jobs are executed
  • user interface: portion of an operating system that handles communication with users, such as shell and graphical user interface (GUI).
  • operating system: collection of software that contrals the overall operation of a computer. It consists of software from application and system.
  • kernel: internal part of an operating system, contains software that perform the very basic functions required by installation.
  • process: execution of a program under the control of the operating system.

Network and Internet

  • LAN: local area network
  • MAN: metropolitan area network
  • WAN: wide area network
  • protocol: rules that a network to function reliably.
  • Ethernet: a set of standards for implementing a LAN with a bus topology.
  • internet: a network of networks.
  • router: connection between networks to form an internet.
  • distributed system: consist of software units that execute as processes on different computers.
  • cluster computing: a distributed system in which many independent computers work closely together to provide computation or services comparable to a much larger computer.
  • grid computing: distributed systems that are more loosely coupled than clusters but that still work together to accomplish large tasks.
  • cloud computing: distributed systems that huge pools of shared computers on the network can be allocated for use.
  • Internet: most notable internet, a worldwide combination of LANs, MANs, and WANs.
  • ISP: Internet Service Providers
  • IP address: Internet Protocol address, a unique identifying address for each computer in an internet-wide addressing system.
  • domain: a "region" of the Internet operated by a single authority. After registrars, a unique domain name will be assigned.
  • domain name system (DNS): system that perform the conversion of mnemonic domanin addresses to IP addresses.
  • File Transfer Protocol (FTP): a client/server protocol for transferring files across the Internet.
  • Hypertext Transfer Protocol (HTTP): a protocol that is used for transferring the hypertext documents between browsers ad web servers.
  • Uniform Resource Locator (URL): a unique address for each document in order to locate and retrieve.
  • Hypertext Markup Language (HTML): system of tags which are special symbols that describe how the document should appear on a display screen.
  • eXtensible Markup Language (XML): a standardized style for designing notational systems for representing data as text files.

Data Abstractions

  • homogeneious array: a "rectangular" block of data whose entries are of same type.
  • heterogeneious array: a block of data whose entries might be of different types.
  • list: a collection whose entries are arranged sequentially. The beginning is called the head and the end is called the tail.
  • stack: a list in which entries are inserted and removed only at the head. The head of a stack is called "top" and its end is called "base" or "bottom". Removing an entry from the top is called "popping an entry". This is a "last-in, first-out" structure.
  • queue a list in which the entries are removed only at the head and new entries are inserted only at the tail. This is a "first-in, first-out" structure.
  • tree: a collection whose entries have a hierarchicalorganization.
  • pointer: a storage area that contains the encoded address.

Database

  • schema: a description of the entire database structure that is used by the database software to maintain the database.
  • Database management system (DBMS): this is separated from the application software.
  • database model: conceptual view of the database, e.g, ralational database model and the object-oriented database model.
  • relation: descprition of data as being stored in rectangular tables. A row in a relation is called tuple. Columns in a relateion are attributes.
  • SQL: Structured Query Language, forms the backbone of most relational database query systems.
